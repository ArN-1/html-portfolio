Git

  • Sistem pelacak modifikasi kode yang biasa digunakan dalam software development. Dalam dunia software development version control system (VCS) merupakan tools yang wajib dikuasai agar memperkecil resiko kegagalan pada saat modifikasi data. Tools ini akan membantu software engineer untuk mengelola perubahan source code dari waktu ke waktu, melacak setiap perubahan kode setiap ada kesalahan tanpa mengganggu team lain yang bekerja dalam satu version control system (VCS).

  • Git merupakan open source version control system gratis yang terdistribusi untuk mengatur segala jenis project dari kecil sampai besar dengan mengandalkan kecepatan dan efisiensi. Git tidak perlu terhubung ke server untuk mendapatkan histori dan menampilkannya cukup membaca langsung dari database local. Dengan integrasi yang dimiliki, Git bisa mengecek isi dari sebuah file yang sudah dilakukan pengecekan. Dan setiap perubahan yang di commit hanya akan ditambahkan ke database Git jadi akan sulit untuk kehilangan data. Ada 3 status pada Git yaitu : ‘committed (data sudah disimpan di database), staged (menandai file yang mengalami perubahan), dan modified (perubahan file yang belum ditandai).

  • Untuk berkomunikasi dengan git bisa menggunakan command line dan hanya perlu instalasi git saja tidak perlu instalasi yang lain.

Installation

  • Installasi Git di Ubuntu

    sudo apt-get install git

  • Cek Installasi

    git --version

  • Konfigurasi awal

     git  config  --global  user.name  “Your  name”
 git  config  --global  user.email  “Your  email"

  • Cek konfigurasi

     git config --list

Untuk mendapatkan repository Git didalam project bisa menggunakan dua cara yaitu dengan git init dan git clone .

git init akan menginisialisasi repository git pada project dan membuat folder .git yang berisi semua file yang dibutuhkan oleh git nantinya.Sedangkan git clone akan menduplikasi project sesuai dengan nama nya dan didalamnya juga akan ada folder .git .
